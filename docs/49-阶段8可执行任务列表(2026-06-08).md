# 阶段 8 可执行任务列表：性能、打包和商业化准备

本文档从 [《阶段 8 缺口清单》](./48-阶段8缺口清单(2026-06-08).md) 进一步拆解为可以直接排期执行的任务列表。

## 任务列表

### 1. 建立性能基线

目标：
- 为索引、搜索、语义、OCR 建立统一基线。

任务：
- 设计 1,000 / 5,000 / 10,000 文档测试集。
- 记录索引总耗时、平均搜索响应时间、内存占用。
- 输出一份可回归对比的 benchmark 结果。

验收：
- 可以重复执行并得到同一套性能报告。

当前进展：
- 已补 `seekmind-benchmark`
- 已支持运行时能力、索引规模、搜索时延和 RSS 基线输出
- 已补 `seekmind-stress`
- 已支持临时隔离 workspace、样本生成、全量索引和搜索探针
- 仍需补真实大库样本、增量索引和恢复压测

### 2. 建立大文档库压测脚本

目标：
- 模拟真实用户大目录场景。

任务：
- 准备压测目录样本或生成器。
- 执行全量索引、增量索引和恢复索引压测。
- 记录失败率和恢复耗时。

验收：
- 能在统一脚本下复现大文档库索引表现。

### 3. 补索引版本迁移机制

目标：
- 让全文索引和向量索引具备版本化升级能力。

任务：
- 设计索引版本元数据结构。
- 在启动时判断索引版本是否兼容。
- 不兼容时触发自动重建或明确提示。

验收：
- 索引结构变化后不会出现静默错乱。

当前进展：
- 已实现全文索引 schema 版本标记与自动重建。
- 已实现语义向量索引 `schema_version` 与失配后清空重建。
- 已在启动流程中接入版本判断与迁移日志。

### 4. 补日志导出

目标：
- 支持用户把日志导出给开发排障。

任务：
- 导出底部日志面板内容到本地文件。
- 补版本号、平台、核心运行时摘要。
- 在 UI 中提供明确入口。

验收：
- 用户可以手动导出日志文件并分享。

当前进展：
- 已在底部日志面板增加导出入口。
- 已支持导出当前事件、运行摘要和环境信息到本地 Markdown 文件。

### 5. 补首次启动引导

目标：
- 降低第一次使用门槛。

任务：
- 识别首次启动状态。
- 引导添加索引目录。
- 引导首次索引和首次搜索。

验收：
- 新用户可以沿主路径完成第一次可用体验。

### 6. 补自动更新基础链路

目标：
- 支持发布后的版本检查与升级入口。

任务：
- 接入 updater 基础配置。
- 先做“手动检查更新”。
- 展示当前版本和可用新版本信息。

验收：
- 用户可以主动检查更新并知道是否有新版本。

### 7. 补隐私说明交付页

目标：
- 让用户清楚理解本地数据、外部模型和日志的边界。

任务：
- 编写隐私说明文案。
- 在设置页或帮助页增加入口。
- 说明本地索引、网络代理、模型请求和日志内容。

验收：
- 用户可以在应用内看到完整隐私说明。

### 8. 补 License / Pro 功能开关骨架

目标：
- 为后续商业化准备最小 feature gate。

任务：
- 定义免费功能和 Pro 功能边界。
- 增加本地 feature flag 结构。
- 在 UI 中支持功能开关或锁定态展示。

验收：
- 应用能够根据本地授权状态展示不同功能范围。

## 推荐执行顺序

1. 性能基线
2. 大文档库压测
3. 首次启动引导
4. 自动更新
5. 隐私说明
6. License / Pro 功能开关
7. 崩溃恢复产品化收口

## 结论

阶段 8 不是单点功能开发，而是一次“发布基础设施收口”。最先应该做的是：
- 指标
- 日志
- 引导与升级入口

没有这三项，后面的更新、引导和商业化都会缺少落地基础。
