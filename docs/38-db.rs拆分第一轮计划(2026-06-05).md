# db.rs 拆分第一轮计划

## 背景

`src-tauri/src/seekmind/storage/db.rs` 已经超过 5000 行，当前同时承担了：

- SQLite 连接与初始化
- schema / migration
- 文档、chunk、block、OCR 任务存储
- 搜索编排与高亮
- 索引任务状态与失败文件
- QA 会话与历史
- 收藏、标签、集合
- 若干通用工具函数与行结构体

这已经不是单一数据库文件，而是一个“上帝对象”。本次拆分目标不是一次性重构成 repository，而是先做物理拆分，保持外部调用方式不变。

## 原则

1. 保留 `crate::seekmind::storage::db::Database` 作为稳定入口。
2. 先拆纯结构体和纯工具函数，再拆 schema 和业务模块。
3. 每一步都能单独编译通过，避免一次性大迁移。
4. 不改外层调用习惯，不引入 trait/repository 重构。

## 推荐目录结构

```text
src-tauri/src/seekmind/storage/
├── db.rs                // 当前门面入口，后续逐步瘦身
├── db/
│   ├── rows.rs          // 所有 FromRow 结构体
│   ├── util.rs          // 时间、路径、scalar count 等工具函数
│   ├── schema.rs        // init_schema / ensure_xxx migration
│   ├── settings.rs      // index_settings / qa_settings / proxy / profiles
│   ├── documents.rs     // documents / chunks / blocks / OCR tasks
│   ├── search.rs        // search / snippet / highlight / rerank
│   ├── task.rs          // current_task / checkpoint / failed_files
│   ├── qa.rs            // qa_history / sessions / messages
│   ├── collections.rs   // collections / collection_items
│   ├── tags.rs          // tags / item_tags
│   └── favorites.rs     // favorites / recent_documents / recent_views
```

> 说明：当前第一轮只做 `rows.rs` 和 `util.rs`，其余模块先不动。

## 第一轮拆分目标

### 1. `rows.rs`

把所有 `sqlx::FromRow` 结构体从 `db.rs` 挪出去，包括：

- `IndexDirRow`
- `DocumentPathRow`
- `FulltextDocumentRow`
- `FulltextChunkRow`
- `SearchRow`
- `DocumentRow`
- `ChunkRow`
- `BlockRow`
- `BlockWithDocumentRow`
- `FailedFileRow`
- `IndexRunSummaryRow`
- `SearchHistoryRow`
- `RecentDocumentRow`
- `RecentViewRow`
- `TagRow`
- `TargetTagRow`
- `FavoriteRow`
- `CollectionRow`
- `CollectionItemRow`
- `QaSettingsRow`
- `NetworkProxySettingsRow`
- `QaModelProfileRow`
- `QaHistoryRow`
- `QaSessionRow`
- `QaMessageRow`
- `IndexCheckpointRow`
- `CurrentTaskRow`

收益：

- 风险最低
- 几乎不改业务逻辑
- 让 `db.rs` 顶部变干净

### 2. `util.rs`

把纯工具函数移出 `db.rs`，优先包含：

- `sqlite_database_path`
- `database_path`
- `normalize_directory_path`
- `is_virtual_directory`
- `is_path_within_dir`
- `default_exclude_dirs`
- `current_unix_ts`
- `format_unix_ts`
- `scalar_count_no_bind`
- `scalar_count_bind`

收益：

- 让数据库门面文件去掉一批散落的 helper
- 后续 `schema.rs` / `task.rs` / `settings.rs` 拆分会更顺

## 拆分顺序

### 第 1 步：`rows.rs` + `util.rs`

只做文件拆分，不改行为。

### 第 2 步：`schema.rs`

把 `init_schema` 和所有 `ensure_xxx_columns()` 挪出。

### 第 3 步：`settings.rs` / `task.rs` / `documents.rs`

按职责拆开高频业务模块。

### 第 4 步：`search.rs` / `qa.rs`

把搜索与问答逻辑进一步收口。

### 第 5 步：`collections.rs` / `tags.rs` / `favorites.rs`

最后收尾低耦合业务模块。

## 第一轮完成标准

- `db.rs` 顶部不再堆叠大量 row struct
- 通用 helper 函数从 `db.rs` 中移出
- `cargo check` 通过
- 外部调用点不变
- 仅做物理拆分，不改变业务行为

## 备注

- 这一轮不拆 repository trait。
- 这一轮不改 `storage/mod.rs` 的外部入口。
- 这一轮先把“地基”整理好，再逐步拆业务。
