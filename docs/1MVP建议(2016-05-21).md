# docMind 第一阶段 MVP

## 1. 第一阶段产品目标

第一阶段只解决一个核心问题：

> 我不记得文件名，但记得文档里大概写过什么，我希望通过关键词找到对应文档和段落。

例如搜索：

```
maven 离线仓库
```

返回：

```
Maven离线构建方案.md

命中段落：
通过 dependency:go-offline 拉取 parent POM、BOM、plugin 依赖，并生成 .offline-repo 目录，用于内网环境构建。
```

用户点击结果后，可以：

```
1. 打开原文件
2. 查看命中段落
3. 查看文件路径
4. 查看所在页码或段落序号
```

## 支持的文档类型

第一阶段建议只支持这些：


暂时不支持：

```

```

原因很简单：第一阶段先保证 **文档解析 + 索引 + 搜索体验** 跑通。

## 3.4 段落切块 Chunk

docMind 的核心不是索引“文件”，而是索引“段落”。

每个文档解析后拆成多个 chunk：

```
chunk 1：标题 + 第一段
chunk 2：第二段
chunk 3：表格文本
chunk 4：PDF 第 2 页中的一段
```

建议规则：

```
Markdown：按标题和空行切
HTML：按 h1/h2/h3/p/li/table 切
DOCX：按段落切
PDF：按页 + 空行切
TXT：按空行切
```

chunk 长度建议：

```
中文 300 ~ 800 字
英文 200 ~ 500 tokens
```

如果段落太短，可以合并相邻段落。

# 7. Rust 核心接口设计

## 7.1 Parser 接口

```Rust
use std::path::Path;
use anyhow::Result;

#[derive(Debug, Clone)]
pub struct ParsedDocument {
    pub title: Option<String>,
    pub file_type: String,
    pub chunks: Vec<DocumentChunk>,
}

#[derive(Debug, Clone)]
pub struct DocumentChunk {
    pub heading: Option<String>,
    pub page_no: Option<u32>,
    pub text: String,
    pub order: usize,
}

pub trait DocumentParser: Send + Sync {
    fn supports(&self, ext: &str) -> bool;

    fn parse(&self, path: &Path) -> Result<ParsedDocument>;
}
```

---

## 7.2 Parser Registry

```Rust
use std::path::Path;
use anyhow::{Result, anyhow};

pub struct ParserRegistry {
    parsers: Vec<Box<dyn DocumentParser>>,
}

impl ParserRegistry {
    pub fn new() -> Self {
        Self {
            parsers: vec![
                Box::new(TxtParser),
                Box::new(MarkdownParser),
                Box::new(HtmlParser),
                Box::new(DocxParser),
                Box::new(PdfParser),
            ],
        }
    }

    pub fn parse(&self, path: &Path) -> Result<ParsedDocument> {
        let ext = path
            .extension()
            .and_then(|v| v.to_str())
            .unwrap_or("")
            .to_lowercase();

        let parser = self.parsers
            .iter()
            .find(|p| p.supports(&ext))
            .ok_or_else(|| anyhow!("Unsupported file type: {}", ext))?;

        parser.parse(path)
    }
}
```

```
本地问答
```

## 最终一句话

docMind 第一阶段不要追求“AI 感”，而是先把这个闭环做到稳定：

```
目录 → 文档 → 文本 → 段落 → 索引 → 搜索 → 定位
```

只要这个闭环打通，docMind 就已经是一个有实际价值的本地文档搜索工具。
