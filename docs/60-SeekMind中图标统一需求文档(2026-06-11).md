# SeekMind 中图标统一需求文档

## 1. 背景
SeekMind 是桌面端知识管理应用，当前产品已经逐步统一了页面布局、卡片层级、按钮样式和深浅色主题。下一步需要统一除 App 图标以外的中图标体系，包括：

- 页面标题图标
- 左侧主菜单图标
- 工具栏按钮图标
- 菜单项图标
- 状态/提示图标
- 卡片内功能图标

目标不是做一批“各自好看”的图，而是做一套可以在全产品复用的图标系统。

## 2. 目标
1. 统一所有中图标的视觉语言，避免同一产品里图标风格混杂。
2. 保证图标在浅色和深色主题下都清晰可辨。
3. 保证图标在 16px、18px、20px、24px、32px 这些常用尺寸下可读。
4. 让图标可以直接映射到产品现有按钮、菜单和标题区，无需二次适配。

## 3. 使用范围
本需求覆盖但不限于以下场景：

- 左侧主导航：搜索、问答、知识库、文档切片、索引状态、设置
- 页面标题区：各页标题前的小图标
- 工具栏：新建、刷新、同步、导出、关闭、展开、折叠、筛选、复制、删除
- 右击菜单：重命名、编辑、设为默认、添加、移除、打开、导出
- 状态提示：成功、警告、错误、信息、空状态
- 卡片内标识：文档类型、来源类型、标签、收藏、最近打开、最近搜索

不包含：

- App 图标
- 大幅插画
- 页面背景装饰图
- 照片级素材

## 4. 设计原则

### 4.1 统一风格
所有图标必须属于同一套视觉系统，不能出现以下混搭：

- 线性图标和拟物图标混用
- 多种线宽混用
- 圆角风、尖角风、扁平风混用
- 填充图标和描边图标随意混用

### 4.2 桌面端原生感
SeekMind 是桌面端应用，图标应偏克制、清晰、实用，避免过度装饰。

- 结构简洁，识别优先
- 不使用复杂渐变
- 不使用高饱和霓虹色
- 不使用卡通化、拟物化、手绘化表达

### 4.3 主题适配
图标必须支持浅色和深色主题。

- 浅色主题下要有足够对比度
- 深色主题下不要发虚、不要糊在背景里
- 需要考虑图标在白底、灰底、深色面板、按钮、菜单中的表现一致性
- 如果图标依赖颜色表达，需要同时给出浅色和深色主题下可直接使用的版本或适配方案

### 4.4 系统一致性
图标不是独立作品，而是 UI 组件的一部分。

- 图标要和按钮、菜单、标题、卡片的视觉密度匹配
- 图标线条粗细要和文字权重协调
- 图标留白要统一，不能一会儿很松一会儿很紧

## 5. 图标风格要求

### 5.1 基础形式
建议采用统一的现代线性图标体系，允许少量实心点缀，但不要大面积填充。

推荐特征：

- 主体以描边为主
- 线宽统一
- 端点和拐角圆润
- 细节少而准
- 轮廓易识别

### 5.2 线宽与圆角
- 统一线宽建议：1.75px 到 2px 的视觉感知
- 图标拐角建议为轻微圆角
- 不要出现过细的发丝线
- 不要出现过粗的黑块

### 5.3 视窗与留白
- 建议以 24x24 或 32x32 画布设计
- 图形主体应居中
- 四周留白均衡
- 不要让笔画贴边
- 不要为了“显大”把图形撑满整个画布

### 5.4 着色方式
- 优先提供单色版
- 单色版应支持按 CSS 颜色继承
- 如需多色版，只允许极少量辅助色
- 不建议依赖渐变作为识别核心

## 6. 图标尺寸要求

### 6.1 常用尺寸
请优先考虑以下成品使用尺寸：

- 16px：菜单、标签、辅助状态
- 18px：页面标题前图标
- 20px：按钮、工具栏
- 24px：卡片头、弹窗内标题
- 32px：少量强调场景

### 6.2 缩放要求
图标在缩放后仍需保持清晰：

- 16px 时不糊成一团
- 18px 时细节不丢失
- 24px 时不显空

## 7. 需要优先统一的图标类别

### 7.1 页面与导航图标
这些图标会长期出现在左侧主导航和页面标题区，要求最高：

- 搜索
- 问答
- 知识库
- 文档切片
- 索引状态
- 设置

### 7.2 动作按钮图标
用于按钮、工具栏、右击菜单：

- 新建
- 编辑
- 保存
- 删除
- 关闭
- 刷新
- 同步
- 导出
- 复制
- 打开
- 过滤
- 收起 / 展开
- 上移 / 下移

### 7.3 内容语义图标
用于卡片、列表、状态与类型标识：

- 文档
- 图片
- Markdown
- PDF
- 文件夹
- 收藏
- 最近搜索
- 最近打开
- 标签
- 成功
- 警告
- 错误
- 信息

## 8. 具体页面使用建议

### 8.1 左侧主菜单
主导航图标要求统一、克制、识别度高。

- 搜索：放大镜
- 问答：对话/消息
- 知识库：书本/集合
- 文档切片：切片/层叠文档
- 索引状态：数据库/状态
- 设置：齿轮

### 8.2 页面标题区
标题前图标建议使用同一套基础图标系统，不要另起炉灶。

- 图标尺寸统一
- 图标底色统一
- 与标题文字的垂直对齐统一

### 8.3 工具栏与菜单项
工具栏图标应偏轻量，避免压迫文本。

- 适合 16px 到 18px 的描边图标
- 常见按钮建议图标+文字组合
- 菜单项图标需要一眼能区分含义

### 8.4 空状态和提示状态
空状态图标需要简洁，不要抢内容。

- 空白、无结果、未选择、加载中、成功、失败都要有统一语言
- 空状态图标可以稍大，但不能复杂

## 9. 交付物要求
请按以下格式交付：

1. 单色 SVG 源文件
2. 可直接用于前端的 SVG 或矢量文件
3. 每个图标独立文件命名
4. 同时提供浅色/深色适配建议
5. 如有必要，提供 16px、20px、24px 三档预览图

### 9.1 命名建议
建议使用统一命名，例如：

- `icon-search`
- `icon-qa`
- `icon-knowledge-base`
- `icon-chunk`
- `icon-index-status`
- `icon-settings`
- `icon-refresh`
- `icon-export`
- `icon-delete`

### 9.2 主题交付
请明确区分浅色和深色主题的可用性：

- 如果是单色 SVG，要求颜色可继承，能直接跟随前端主题 token 变色
- 如果是带颜色图标，必须提供浅色 / 深色两套可用版本，不能只给一套在另一主题里发白或发黑的图
- 交付时请附一份浅色主题预览和一份深色主题预览
- 图标最终效果以 SeekMind 当前浅色 / 深色界面实机截图为准

## 10. 验收标准
图标交付后，按以下标准验收：

- 同类图标在同一页面里看起来属于同一体系
- 深色主题下不会出现白底、发灰、发虚、过细的问题
- 16px 尺寸下仍可识别
- 与文字、按钮、菜单项搭配时不突兀
- 图标不需要针对每个页面单独微调
- 前端可以直接复用，不需要重新绘制

## 11. 备注
当前产品已经在 UI 层面逐步统一：

- 全局主题 token
- 卡片层级
- 选中态
- 输入框和下拉框
- toast 提示

因此图标也要遵循同样的统一原则，不能只在单个页面看起来好看，而是要在整个 SeekMind 中长期稳定复用。
