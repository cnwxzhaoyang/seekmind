# 设置页瘦身与历史记录归口方案

## 背景

设置页目前承载了索引规则、语义搜索、问答模型、嵌入模型、外观、隐私、危险操作等内容。其中“来源问答”区域还包含问答历史列表。问答历史会持续增长，导致设置页高度被不断拉升，也让“低频配置”和“高频数据管理”混在一起。

## 问题

- 设置页内容过多，扫描成本高。
- 问答历史属于持续增长的数据，不适合放在设置页。
- 问答页已经有会话列表、重命名、删除、复制、导出等管理能力，设置页重复展示历史会造成入口混乱。
- 历史列表高度不可控，会破坏设置页布局稳定性。

## 设计原则

- 设置页只放低频配置项。
- 会增长的数据归口到专门页面或管理入口。
- 设置页可以提供跳转入口，但不直接承载完整列表。
- 历史、会话、最近记录等列表必须分页、限高或移动到独立页面。

## 调整方案

### 1. 设置页保留内容

- 索引规则
- 语义搜索参数
- 问答模型配置
- 嵌入模型配置
- 外观和语言
- 隐私说明
- 危险维护操作

### 2. 从设置页移除内容

- 问答历史完整列表
- 问答历史单条删除按钮
- 随历史数量增长的问答记录卡片

### 3. 问答区域改造

设置页“来源问答”区域只保留：

- 是否启用
- provider / model / base url / api key
- 温度、输出 token、上下文参数、证据阈值
- 保存设置
- 刷新配置
- 测试连接
- 打开问答会话入口
- 隐私提示与连接测试结果

### 4. 问答历史归口

问答历史和会话管理统一归口到 `/qa`：

- 左侧会话列表显示历史会话
- 右键菜单管理重命名、删除、复制 Markdown、导出 Markdown
- 多轮消息与来源详情在问答页查看

### 5. 后续可选增强

- 新增“数据管理”页面，集中放搜索历史、最近文档、收藏、问答会话清理、数据库统计。
- 在设置页危险区域增加折叠式“高级维护”，放清空问答会话、清空搜索历史等动作。
- 问答页增加会话分页或虚拟列表，避免会话很多时左侧列表性能下降。

## 本次实施范围

- 移除设置页问答历史列表。
- 移除设置页问答历史删除逻辑。
- 设置页问答区域增加“打开问答会话”入口。
- 保留现有问答模型配置与测试连接能力。
