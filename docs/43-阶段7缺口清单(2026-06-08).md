# 阶段 7 缺口清单：OCR 和更多格式支持

本文档是 [《阶段 7 对照表》](./42-阶段7对照表(2026-06-08).md) 的配套清单，聚焦当前尚未完全闭环的能力点。

## 已确认缺口

### 1. 扫描版 PDF 检测闭环

现状：
- 代码里已经有 PDF OCR 队列重跑、OCR 结果落库和运行时状态提示。
- 但没有看到独立的“扫描版 PDF 检测”分流逻辑，把可复制文本 PDF 与扫描版 PDF 做出明确判定。

缺口：
- 缺少自动识别扫描版 PDF 的判定逻辑。
- 缺少“检测 -> 入队 -> OCR -> 回填”的完整闭环说明。

建议：
- 先补一个稳定的扫描版 PDF 判断规则，再把 OCR 任务接到该判定结果上。

### 2. 图片文字识别主链路接入

现状：
- 已有 macOS Vision OCR helper。
- 已有语言探测和运行时状态暴露。

缺口：
- 图片 OCR 还没有进入索引主链路。
- 没看到图片文件在索引时自动走 OCR 并回填到 blocks / chunks / search 的完整链路。

建议：
- 先支持 PNG / JPG / JPEG 等基础图片格式的 OCR 入库，再考虑其他图像格式。

### 3. XLSX 文本提取

现状：
- 阶段 7 规划里明确要求支持 XLSX。
- 现有扫描器支持列表和解析分支里没有 `xlsx`。

缺口：
- 没有 XLSX 专门的解析路径。
- 也没有 CSV/表格行列到文本/块的映射策略。

建议：
- 优先做“提取表格为顺序文本”的基础版，不追求表格结构复原。

### 4. EPUB 支持

现状：
- 规划要求支持 EPUB。
- 当前代码中没有 EPUB 解析分支。

缺口：
- 缺少 EPUB 解包、正文 HTML 抽取和章节串联策略。

建议：
- 优先做章节级文本抽取，先让内容可搜索，再考虑更细粒度预览。

### 5. RTF 支持

现状：
- 规划要求支持 RTF。
- 当前代码中没有 RTF 解析分支。

缺口：
- 缺少 RTF 到纯文本的转换链路。

建议：
- 先接系统工具或成熟转换器，保证能抽文本即可。

### 6. 格式支持状态说明

现状：
- 已有 parser runtime / PDF OCR / Office 转换的状态信息。
- 前端能看到部分提示，但没有完整的“格式支持矩阵”。

缺口：
- 没有统一展示各格式支持情况、降级路径和限制说明。
- 用户难以直接判断“当前哪些格式可用、哪些会降级、哪些不可用”。

建议：
- 在状态页或设置页增加一块“格式支持状态”说明，列出支持范围与降级策略。

## 推荐补齐顺序

1. `XLSX`
2. `EPUB`
3. `RTF`
4. 图片 OCR 主链路
5. 扫描版 PDF 检测闭环
6. 格式支持状态说明

## 结论

阶段 7 当前不是未做，也不是做完，而是处于“核心 OCR 已有、格式扩展缺口仍然存在”的状态。

如果后续要继续推进，建议先补最容易影响用户感知的格式支持，再补扫描版 PDF 的检测闭环和状态说明。
