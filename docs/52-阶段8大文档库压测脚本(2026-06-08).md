# 阶段 8 大文档库压测脚本（2026-06-08）

本文档对应阶段 8 的第 2 项任务：建立大文档库压测脚本。

当前已经新增 `seekmind-stress` 命令，它会：

- 生成临时压测目录
- 创建 1,000 / 5,000 / 10,000 三档样本
- 调用现有索引全量重建链路
- 记录索引耗时、搜索探针耗时、峰值 RSS
- 输出 JSON 报告

## 1. 命令入口

```bash
npm run benchmark:stress -- --output docs/reports/seekmind-stress.json --cleanup
```

或：

```bash
cargo run --manifest-path src-tauri/Cargo.toml --bin seekmind-stress -- --output docs/reports/seekmind-stress.json --cleanup
```

## 2. 默认行为

默认会生成：

- 1,000 文档
- 5,000 文档
- 10,000 文档

每个样本会创建若干分目录，文件类型会在 `md / txt / log / json / csv / html` 间轮换。

## 3. 输出内容

报告 JSON 里会包含：

- `size`
- `created_files`
- `directories`
- `rebuild_ms`
- `search_ms`
- `max_rss_mb`
- `indexed_docs`
- `indexed_chunks`
- `failed_files`
- `pdf_ocr_tasks`
- `query_hits`

## 4. 隔离方式

为了避免污染用户真实数据，这个脚本会通过环境变量切到临时根目录：

- `SEEKMIND_DATA_ROOT`
- `SEEKMIND_CACHE_ROOT`

这样数据库和全文索引都会落到压测 workspace 里。

## 5. 当前定位

这仍然是第一版压测脚本，覆盖的是：

- 样本生成
- 全量索引
- 搜索探针
- 粗粒度内存采样

还没有覆盖：

- 增量索引压测
- 恢复索引压测
- 长时间持续压测
- 失败率统计分层

这些会作为下一步增强。
