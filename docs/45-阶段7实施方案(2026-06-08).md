# 阶段 7 实施方案：OCR 和更多格式支持

本文档用于说明阶段 7 当前的实现结果，以及后续如果继续补“自动检测闭环”应如何推进。

## 1. 当前状态

阶段 7 的最低实施目标已经完成，当前代码库已经具备：
- 扫描版 PDF 可手动 OCR，并在完成后参与搜索
- 图片 OCR 已进入索引主链路
- `XLSX / EPUB / RTF` 已进入索引与搜索链路
- 状态页已提供“格式支持状态”说明

因此，阶段 7 已经从“能力补齐阶段”进入“体验增强阶段”。

## 2. 当前保留的增强项

### 2.1 扫描版 PDF 自动检测闭环

目标：
- 让系统自动识别扫描版 PDF，而不是要求用户自己判断。

建议推进方式：
- 先做轻量检测，不直接自动入队。
- 规则稳定后，再补自动入队 OCR。
- 最终补成“检测 -> OCR -> 回填 -> 搜索”的完整闭环。

## 3. 推荐实现路径

### 3.1 检测层

建议统一输出：
- 是否为扫描版 PDF
- 是否已有可提取文本
- 当前是否可自动进入 OCR

### 3.2 任务层

复用现有 OCR 任务体系：
- OCR 重跑
- OCR 进度事件
- OCR 结果回填

新增重点：
- 自动检测触发条件
- 自动入队策略
- 失败后的回退与提示

### 3.3 状态层

建议继续沿用当前状态页的展示方式，增加：
- 扫描版 PDF 自动检测状态
- 自动 OCR 是否启用
- 最近自动检测结果说明

## 4. 风险点

1. PDF 自动检测规则不稳定时，容易误判文本型 PDF。
2. 自动入队 OCR 可能增加本地计算开销。
3. 用户需要明确知道“支持 OCR”与“自动 OCR”之间的区别。

## 5. 当前验收结论

阶段 7 最低标准已经满足：
- 扫描版 PDF 可手动 OCR，完成后可搜索。
- 图片文件可 OCR 并参与搜索。
- `XLSX / EPUB / RTF` 可进入索引并搜索。
- 状态页能明确展示支持范围和降级策略。

因此当前不再建议把阶段 7 判断为“未完成”，更准确的判断是：
- `阶段 7：已完成`
- `扫描版 PDF 自动检测闭环：后续增强`

## 6. 后续建议

如果后续继续推进，建议只保留一个任务方向：
1. 扫描版 PDF 自动检测闭环

其余阶段 7 原始功能项可以视为已收口完成。
