# 阶段 7 实施方案：OCR 和更多格式支持

本文档用于指导阶段 7 的后续实现，目标是把 OCR 和多格式支持补成一个可交付的闭环，而不是零散增加几个入口。

## 1. 实施目标

阶段目标：
- 扩大 SeekMind 可索引资料范围。
- 让扫描版 PDF、图片、PPT、Excel、EPUB、RTF 等资料逐步进入搜索范围。

核心原则：
- 先稳定，再扩展。
- 先可搜索，再优化预览。
- 先文本抽取，再考虑版面还原。
- OCR 默认不全量开启，必须保持可控、可解释、可回退。

## 2. 推荐架构

### 2.1 解析入口分层

建议把解析能力拆成三层：
- 基础文本解析层：txt、md、html、docx、doc、ppt、pptx 等。
- OCR 增强层：扫描版 PDF、图片 OCR。
- 格式扩展层：xlsx、epub、rtf 等新格式。

### 2.2 能力探测层

需要统一暴露：
- 当前支持格式
- 当前降级策略
- OCR 可用语言
- PDF OCR 是否可用
- Office 转换器是否可用

### 2.3 任务处理层

建议保留两类任务：
- 索引任务：常规文件扫描、解析、切块、写库、建索引。
- OCR 任务：PDF OCR 重跑、图片 OCR 入库、失败重试。

## 3. 实施路径

### 3.1 先补格式扩展

优先实现：
- `xlsx`
- `epub`
- `rtf`

原因：
- 这些格式的收益明确，链路相对独立。
- 完成后可以立刻扩大可搜索文档覆盖面。

### 3.2 再接图片 OCR

实现顺序：
- 先让 PNG / JPG / JPEG 可识别。
- 再决定是否扩展到 HEIC、WEBP 等格式。

要求：
- OCR 结果必须回写到索引数据。
- 搜索必须能命中 OCR 文本。
- 失败必须明确提示。

### 3.3 再补扫描版 PDF 检测闭环

实现顺序：
- 判定扫描版 PDF。
- 自动或手动进入 OCR 任务。
- OCR 结果回填并参与全文搜索。

要求：
- 不追求版面还原。
- 不做表格深度理解。
- 必须能看到任务状态、耗时和失败原因。

### 3.4 最后补格式支持状态说明

建议展示：
- 支持格式列表
- 当前可用能力
- 当前降级路径
- 当前限制

建议位置：
- 状态页
- 或设置页的“运行时能力”区域

## 4. 数据与状态建议

建议至少保留以下状态字段：
- 支持格式列表
- OCR 语言列表
- PDF OCR 可用性
- Office 转换器可用性
- 当前格式降级说明
- OCR 任务总数

## 5. 风险点

1. XLSX 和 EPUB 的解析质量可能低于纯文本格式。
2. 图片 OCR 会增加本地计算成本。
3. 扫描版 PDF 识别会受到图像质量和语言模型包体积影响。
4. 如果状态说明不清晰，用户容易误判“支持”与“可搜索”的边界。

## 6. 验收标准

阶段 7 完成的最低标准：
- 扫描版 PDF 可手动 OCR，完成后可搜索。
- 图片文件可 OCR 并参与搜索。
- XLSX / EPUB / RTF 可进入索引并搜索。
- 状态页能明确展示支持范围和降级策略。

## 7. 建议落地顺序

1. 格式扩展：XLSX / EPUB / RTF
2. 图片 OCR 主链路
3. 扫描版 PDF 检测闭环
4. 格式支持状态说明

## 8. 结论

阶段 7 的后续推进重点不是再加“入口”，而是把现有 OCR 能力和格式扩展真正接入统一索引链路，并让用户清楚看到能力边界。
