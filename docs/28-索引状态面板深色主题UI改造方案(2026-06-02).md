# 索引状态面板深色主题 UI 改造方案

## 目标

将 `src/pages/StatusPage.vue` 的 UI 改造为 `ui-prototype/index_status/IndexStatusPanel.vue` 原型中的**深色主题两栏卡片式布局**，保留现有 script 层全部逻辑（API 调用、轮询、事件监听、拖拽导入、右键菜单等）。

## 改造范围

| 文件 | 变更 |
|------|------|
| `src/pages/StatusPage.vue` | 重写 `<template>` 和 `<style>`，保留 `<script setup>` 全部逻辑 |
| `src/locales/zh-CN.json` | 追加新 UI 段落所需的国际化 key |
| `src/locales/en.json` | 追加新 UI 段落所需的国际化 key |

## 设计要点

### 3.1 深色主题配色（GitHub 暗色模式）

| 元素 | 色值 | 用途 |
|------|------|------|
| 页面背景 | `#0d1117` | 主背景 |
| 卡片背景 | `#161b22` | 所有卡片背景 |
| 分隔线/边框 | `#30363d` | 边框、分割线 |
| 主文字 | `#e6edf3` | 标题、主要数值 |
| 次要文字 | `#8b949e` | 标签、描述文本 |
| 辅助文字 | `#6e7681` | 描述说明 |
| 成功状态 | `#4caf50` (绿) | 状态徽章、成功统计 |
| 错误状态 | `#f85149` (红) | 错误数、失败状态、错误图表 |
| 进行中/链接 | `#58a6ff` / `#1f6feb` (蓝) | 进度条、链接按钮、文件状态 |
| 警告/跳过 | `#ffc857` (黄) | 跳过统计、待处理图标 |

### 3.2 两栏七卡片布局

```
┌──────────────────────────────────────────────────────────────┐
│  📚 索引状态                         [● Python 索引中] 🕐 最后更新 🔄 │  ← header
├──────────────────────┬───────────────────────────────────────┤
│ 左栏 (1fr)          │ 右栏 (1fr)                            │
│                      │                                       │
│ ⚙️ 索引控制          │ ℹ️ 索引信息 (目录树改造)               │
│  [⏸暂停][▶️恢复][🔄重新索引] │  目录名    文档数  段落数      │
│                      │                                       │
│ 📊 统计信息 (2×2)    │                                       │
│  📄 已索引文件  N    │  📁 已扫描文件  N                    │
│  ⚠️ 错误数     N     │  📋 待处理    N                      │
│                      │                                       │
│ ⏱️ 索引进度           │ ⚠️ 最新异常                           │
│  ████████░░ 60%      │  ┃ 错误文件：xxx                      │
│  正在索引：xxx.pdf   │  ┃ [解析错误]                         │
│  ⏰ 耗时 --:--:--    │  ┃ 异常时间：--                       │
│  📅 开始时间 --      │  ┃ 错误信息：xxx                      │
│  ✓成功 N ✕失败 N ⊘跳过 N │  ┃ 堆栈信息：xxx                 │
│                      │  [查看所有异常 →]                     │
│ 🔴 错误摘要           │                                       │
│  [SVG 圆环图]        │                                       │
│  错误类型分布条形图   │                                       │
│  Python版本 / 超时   │                                       │
└──────────────────────┴───────────────────────────────────────┘
```

### 3.3 关键 UI 元素

- **状态脉冲绿点**：`@keyframes pulse` 动画，2s 无限呼吸
- **渐变进度条**：`linear-gradient(90deg, #1f6feb, #58a6ff)`
- **SVG 错误圆环图**：120px，`stroke-dashoffset` 实现环形填充
- **按钮样式**：深色背景 `#0d1117`，hover 蓝色高亮；primary 按钮 `#1f6feb`
- **响应式断点**：1400px 变单栏，768px 垂直堆叠

## 数据映射

| 原型字段 | 数据来源 | 备注 |
|----------|----------|------|
| `indexStatus` | `taskDisplayState.label` + `parserRuntime.active` | 如"Python 索引中" |
| `lastUpdateTime` | `new Date().toLocaleString()` | 实时更新 |
| `statistics.indexedFiles` | `status.value?.indexed_docs ?? 0` | |
| `statistics.indexedCount` | `status.value?.scanned_docs ?? 0` | |
| `statistics.errorCount` | `status.value?.failed_files ?? 0` | |
| `statistics.pendingFiles` | `Math.max(task.total - task.scanned, 0)` | |
| `progress.overall` | `indexProgressPercent` | 已存在 |
| `progress.currentFile` | `task.current_file ?? '-'` | |
| `progress.duration` | `"--:--:--"` | 占位符，后续加字段 |
| `progress.startTime` | `"--"` | 占位符，后续加字段 |
| `progress.success` | `task.succeeded ?? 0` | |
| `progress.failed` | `task.failed ?? 0` | |
| `progress.skipped` | `task.skipped ?? 0` | |
| `errorTypes[]` | 从 `failedGroups` 按 `code`/`category` 聚合 | |
| `environment.pythonVersion` | 从 `parserRuntime?.python_bin` 提取 | |
| `environment.timeout` | `parserRuntime?.timeout_ms + ' ms'` | |
| `indexedFiles[]` | 使用目录树 `visibleDirRows` | 文件列表区域改造为目录列表 |
| `exceptions[]` | 从 `status.value?.failed_items` 构造 | |

## 实施步骤

1. **添加新 locale 键**：在 `zh-CN.json` / `en.json` 添加原型所需文案
2. **重写 `<template>`**：按原型结构重建，数据绑定替换为现有响应式变量
3. **添加 `<style scoped>`**：移植原型 CSS，移除 Tailwind 浅色类
4. **调整脚本**：新增 `lastUpdateTime` + `errorTypes` 计算属性等微调
5. **验证**：`npm run dev` / build

## 变更说明

- Parser Status 信息合并到错误摘要卡的 `environment-info` 区域
- Incremental Summary 数据合并到进度卡的 `status-stats` 区域
- 文件列表区域使用目录树 `visibleDirRows` 展示目录维度的列表
- 右键菜单、拖拽导入等功能完整保留
