# 61-设置面板信息架构优化(2026-06-12)

## 背景

当前设置页存在几个明显问题：

1. `索引规则` 与 `操作说明` 都在解释索引行为，信息重复。
2. `语义检索` 与 `语义模型` 分离，用户很难建立“检索能力依赖 embedding 可用性”的认知。
3. `来源问答` 命名不准确，实际承载的是 LLM 连接与回答参数。
4. 语义参数缺少解释，用户只能看到数字，无法理解调节影响。

## 目标

1. 删除重复说明，设置页只保留“可操作配置”。
2. 将语义检索参数与向量模型状态放到同一组里。
3. 统一用户语言，减少“启用但不可用”之类的状态误导。
4. 给专业参数补充轻量级帮助提示，不增加额外说明卡片。

## 结构调整

### 1. 索引规则

原结构：

- 索引规则
- 操作说明

调整后：

- 将 `索引规则` 更名为 `索引过滤`
- 删除 `操作说明` 卡片
- 仅保留：
  - 排除目录
  - 排除扩展名
  - 最大文件大小

说明：

- 支持的文件类型、索引目录来源等静态说明不再放在设置页重复展示
- 文件类型说明统一留在 `索引状态`

### 2. 语义相关

原结构：

- 语义检索
- 来源问答
- 语义模型

调整后：

- 将 `语义检索` 与 `语义模型` 改为同一组连续区块
- 导航中只保留一个 `语义检索`
- `语义检索` 头部状态按组合状态展示：
  - 已关闭：用户关闭语义检索
  - 已启用：用户开启且 embedding 可用
  - 不可用：用户开启但 embedding 不可用

### 3. LLM 连接

原命名：

- 来源问答

调整后：

- 更名为 `LLM 连接`

说明：

- 这一块主要是 provider、base URL、API key、model、回答参数与连接列表
- “来源问答”无法准确表达其真实职责

## 参数帮助策略

对语义检索中的专业参数，在标签右侧增加 `info` 图标。

交互：

- 桌面端 hover 显示 tooltip
- tooltip 内容使用一到两句话
- 不再增加额外说明段落或说明卡片

建议提示文案：

- `语义权重`
  - 语义相似度在最终排序中的占比，越高越偏向向量召回结果。
- `语义阈值`
  - 低于该相似度的候选不会参与最终排序。
- `结果偏好`
  - 控制收藏、最近打开、搜索历史等行为是否参与排序加分。
- `排序权重`
  - 控制标题、文件名、偏好信号在最终得分中的影响强度。
- `标题权重`
  - 标题命中对最终排序的影响程度。
- `文件名权重`
  - 文件名命中对最终排序的影响程度。
- `偏好权重`
  - 收藏、最近打开、搜索历史等偏好信号对最终排序的影响程度。

## 实施范围

本次实现包含：

1. 删除设置页 `操作说明`
2. `索引规则` 更名为 `索引过滤`
3. `语义检索` 与 `语义模型` 合并为同一组连续区块
4. `来源问答` 更名为 `LLM 连接`
5. 为语义检索相关参数增加帮助 tooltip
6. 语义检索头部状态改为依赖 embedding 可用性

本次不包含：

1. 后端设置结构调整
2. 语义参数定义变更
3. 状态页与设置页之间的内容合并
