# 国际化（i18n）实施方案

## 背景

当前 docMind 所有 UI 文本均为硬编码简体中文，分布在 16 个 Vue 文件中。项目需要支持中英文切换，为后续多语言用户做准备。

## 目标

1. 将全部 UI 文本从硬编码改为 i18n 翻译调用。
2. 支持中文（zh-CN）和英文（en）两套语言。
3. 自动检测系统语言，支持在设置页手动切换且持久化偏好。

## 技术选型

使用 `vue-i18n`（latest v10 for Vue 3）作为国际化方案。

理由：
- Vue 3 官方推荐，template 中可直接用 `$t()`。
- script 中通过 `useI18n().t()` 调用，支持 Composition API。
- 原生支持 named interpolation（`第 {page} 页`）、pluralization、linked messages。
- 社区成熟，生态完善。

预计总量 **180 ~ 220 个 key**。

## Script 段中的中文处理策略

有四类特殊情况需额外处理：

1. **`window.confirm()`** — 无法直接使用 `$t()`，改用 `useI18n().t()` 传入。
2. **`formatDocmindError(error, "中文fallback")`** — 第二个参数改为 `t('key')`。
3. **`infoMessage.value = "已添加目录..."`** — 改为模板字符串嵌入翻译调用。
4. **`scopeMeta`、`items` 等 data 中的 label** — 提取为 computed 属性，动态调用 `t()`。

## 语言切换方案

### 自动检测

优先读取 `navigator.language`：
- 匹配 `zh-CN` / `zh` → 加载中文
- 其余 → 加载英文（fallback locale）

### 手动切换

在设置页（SettingsPage）新增一个「界面语言」选择器（下拉框），值存入 `localStorage`，覆盖自动检测结果。

### 持久化

语言偏好写入 `localStorage`，key 为 `docmind-locale`。i18n 初始化时优先检查 localStorage，其次回退到系统检测。

## 实施步骤

1. **安装依赖**：`npm install vue-i18n@latest`
2. **创建 i18n 基础设施**：`src/i18n/index.ts` + `zh-CN.json` + `en.json`
3. **注册 i18n 插件**：修改 `src/main.ts`
4. **按从内到外顺序替换组件**：
   - 纯展示组件：TaskCard → LogPanel → SearchResultCard → SearchResultGroupCard
   - 功能组件：SemanticPanel → Sidebar
   - 页面组件：SettingsPage → StatusPage → LibraryPage → ChunksPage → AppSearch
5. **处理 script 中的特殊字符串**：confirm / info / error / fallback 消息
6. **设置页添加语言开关**
7. **更新 index.html 的 lang 属性**
8. **验证**：切换语言后所有文本正确渲染

## 验收标准

1. 系统语言为中文时自动显示中文，英文时自动显示英文。
2. 设置页可手动切换语言，刷新后保持选择。
3. 所有页面的标题、按钮、标签、placeholder、空状态、toast 消息均随语言切换。
4. `window.confirm` 弹窗内容正确翻译。
5. `formatDocmindError` 的 fallback 文本正确翻译。
6. 不存在遗漏的硬编码中文。
