## 一、当前页面主要问题

### 1. 左侧栏太“后台系统化”

现在左侧有：

搜索 / 问答 / 知识库 / 文档切片 / 索引状态 / 设置  
索引目录  
最近搜索  
最近打开  
收藏结果

信息很全，但视觉上每个区域都像一个独立卡片，导致左侧很碎。对于桌面 App 来说，左侧更适合做成 **macOS Finder 风格导航栏**，弱化边框，强化选中态。

建议：

* 左侧背景统一为浅灰蓝。
* 模块之间减少卡片边框，用标题 + 列表分组即可。
* 当前选中项用柔和蓝色背景。
* 数字徽标弱化，不要每个目录都显示两三个数字。
* “最近搜索 / 最近打开 / 收藏结果”可以折叠，默认只展示一个。

---

### 2. 中间结果列表太像“日志列表”

现在每个结果卡片信息很多：

文件名、路径、标题路径、DOC、文档聚合、命中片段、总分、排序依据、最近打开、段数、百分比等。

问题是：**所有信息都在抢注意力**。用户真正关心的是：

1. 这是哪个文档？
2. 为什么命中？
3. 命中内容在哪里？
4. 是否值得点开？

建议把结果卡片压成三层：

```
第一层：文件名 + 文件类型 + 匹配分数
第二层：路径 / 标题路径
第三层：命中摘要，高亮关键词
底部：命中段落数 / 更新时间 / 操作按钮
```

“文档聚合、总分、排序依据”这些可以放进 hover 或右上角 tooltip，不要常驻显示。

---

### 3. 匹配分数显示方式不自然

现在右侧显示 524%、517%、365%，这个对用户不直观，而且容易让人觉得算法有 bug。

建议改成：

```
相关度 98
相关度 95
相关度 86
```

或者：

```
高度相关
较高相关
可能相关
```

内部仍然可以用你的综合分，但 UI 层最好归一化到 0-100。

---

### 4. 右侧详情区信息过重

右侧现在展示：

文件标题  
路径  
标题路径  
标签  
段落序号  
命中段落  
上下文预览  
上一段  
当前段

这个信息结构是对的，但视觉上有点像“表单详情页”，不够像阅读器。

建议把右侧改成：

```
顶部：文档信息卡
中部：命中段落阅读区
底部：上下文导航
```

当前段落应该是视觉中心，可以做成更强的浅蓝背景块。上一段 / 下一段弱化为灰色背景。

## 六、我建议你优先改这 8 个点

1. **把左侧栏卡片化改成 Finder 风格分组导航。**
2. **搜索结果卡片减少 30% 信息量。**
3. **匹配分数从 524% 改成 0-100 相关度。**
4. **路径、标题路径统一变成小号灰色文本。**
5. **只突出关键词、文件名、当前选中项。**
6. **右侧详情改成阅读预览，不要像字段详情页。**
7. **底部日志栏默认收起，只保留状态摘要。**
8. **减少边框，多用留白、字号、字重区分层级。**

## 七、整体目标

你现在的页面是：

> 功能完整的工程调试界面。

应该调整成：

> 面向普通用户的本地智能文档搜索工具。

也就是说，用户打开后第一眼应该觉得：

```
我可以搜索。
我知道哪个结果最相关。
我可以快速看到命中的原文。
我可以马上打开这个文件。
```

而不是第一眼看到大量字段、分数、标签、路径和技术状态。

这个页面不需要大改交互，主要是**视觉降噪 + 信息层级重排**。如果你继续用 Vue3 + Tailwind，这一版重构成本不会太高。
