# 索引状态当前文件实时展示方案

## 问题描述

在"索引状态"页面的"当前任务"区域中，用户无法直观看到当前正在解析的文档。目前仅显示了上一个解析完成的文档的进度，缺少正在解析的文件名和阶段信息。

## 根因分析

`current_file` 在数据库和后端轮询响应中**正确设置**，但前端展示方式导致用户无法感知：

| 数据 | 值（示例） | 位置 | 问题 |
|------|-----------|------|------|
| `label` | "正在重新索引本地文档" | 主描述（最醒目） | 无文件信息 |
| `details` | "正在提取内容块" / "正在生成切片 · section3" | 页脚（小字） | 无文件名 |
| `current_file` | "/Users/.../example.pdf" | 指标小卡片（截断） | 不醒目 |

**关键链路**：
- 解析文档前，`set_current_task` 将 `details` 设置为文件完整路径（indexer.rs:461）
- 解析过程中，`spawn_parser_progress_forwarder` 将 `details` 覆盖为解析阶段消息（如 "正在提取内容块"），但**没有包含文件名**
- 前端 `DocMindTaskCard` 的主描述固定为 `label` = "正在重新索引本地文档"，`details` 只出现在卡片页脚

## 修改方案

### 第1处：后端 — `spawn_parser_progress_forwarder`

**文件**：`src-tauri/src/docmind/storage/indexer.rs`

**位置**：`spawn_parser_progress_forwarder` 函数内部，约第751行

**当前代码**：

```rust
let details = parser_event_message(&event, &fallback_message);
```

**改为**：

```rust
let file_name = std::path::Path::new(&path)
    .file_name()
    .and_then(|n| n.to_str())
    .unwrap_or(&path);
let details = format!("{} — {}", file_name, parser_event_message(&event, &fallback_message));
```

**效果**：`details` 从 `"正在提取内容块"` 变为 `"example.pdf — 正在提取内容块"`。该字段会出现在：

- `DocMindTaskCard` 页脚（`{{ props.task.details }}`）
- 下文第2处修改后的主描述区域

### 第2处：前端 — StatusPage.vue

**文件**：`src/pages/StatusPage.vue`

**位置**：`DocMindTaskCard` 组件的 `description` prop，约第953行

**当前代码**：

```vue
:description="status.current_task.label"
```

**改为**：

```vue
:description="status.current_task.details"
```

**原理**：`DocMindTaskCard` 中 `displayDesc` 计算逻辑为：

```ts
const displayDesc = computed(
  () => props.description || props.task?.details || t("taskCard.defaultDesc")
);
```

- `props.description` = `status.current_task.details`（新值）
- 当 `details` 为空时（极短暂间隙），自动回退到 `taskCard.defaultDesc` 默认文案
- 无需额外判断逻辑

### 影响范围

| 阶段 | 修改前 details | 修改后 details（页脚 & 主描述） |
|------|---------------|------------------------------|
| 删除文件 | "清理已删除文件" | "清理已删除文件" |
| 开始解析 | 文件完整路径 | "example.pdf — 正在解析文档" |
| 提取阶段 | "正在提取内容块" | "example.pdf — 正在提取内容块" |
| 切片阶段 | "正在生成切片 · 第三节" | "example.pdf — 正在生成切片 · 第三节" |
| 解析完成 | "解析完成" | "example.pdf — 解析完成" |

- 删除阶段不受影响（不走 `spawn_parser_progress_forwarder`）
- 暂停/恢复阶段不受影响
- 单文档拖拽刷新不受影响（使用独立事件通道）

### 验证方式

1. 触发全量重新索引（点击"重新索引"按钮）
2. 观察"当前任务"卡片的主描述区域是否实时显示当前解析的文件名
3. 观察页脚是否同步显示文件名 + 阶段信息
4. 验证文件切换时描述是否正确更新
5. 验证解析完成后卡片消失
