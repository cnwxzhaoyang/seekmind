# SeekMind 拖拽导入与虚拟目录方案

本文档用于定义 `文档目录` 页的拖拽导入规则，解决当前只能通过“添加目录”导入索引范围的问题。

目标是让用户可以直接拖入：

```text
1. 目录
2. 单个文档
3. 多个文档
```

同时保证目录树仍然清晰可读，不会因为拖入少量文件而误把整棵父目录树纳入索引。

---

## 1. 背景问题

当前 `文档目录` 页仅支持点击“添加目录”导入。

这会带来几个问题：

```text
1. 选择目录的操作成本偏高。
2. 单文件快速导入不方便。
3. 当用户只想补几个文档时，很容易被迫手动找目录。
4. 目录树的组织方式还没有和导入方式打通。
```

---

## 2. 改造目标

```text
1. 支持拖拽导入目录。
2. 支持拖拽导入单个或多个文档。
3. 目录树按层级组织，不再只是平铺列表。
4. 单文件导入时，不误把整棵父目录树纳入索引。
5. 对“无法归入真实目录树的单文件”提供虚拟目录承载。
```

---

## 3. 核心规则

### 3.1 拖入目录

```text
1. 目录直接作为真实索引目录加入。
2. 目录下的子目录仍然按树展示。
3. 目录的导入行为保持和当前“添加目录”一致。
```

### 3.2 拖入单个文档

```text
1. 先判断文件所在目录是否已存在于当前目录树中。
2. 如果存在，则归入真实目录树对应节点。
3. 如果不存在，则归入虚拟目录。
4. 不自动把该文件父目录整棵子树纳入索引。
```

### 3.3 拖入多个文档

```text
1. 每个文档独立判断归属。
2. 已存在于真实目录树中的文件，归入真实目录。
3. 不在真实目录树中的文件，归入虚拟目录。
4. 同一来源父目录的文件可以在虚拟目录下自动分组。
```

---

## 4. 虚拟目录设计

### 4.1 虚拟目录的作用

虚拟目录用于承载“单文件导入但其真实父目录并不在当前索引树中”的内容。

典型场景：

```text
1. 用户从桌面拖了一个 md 文件。
2. 用户从下载目录拖了几个 docx 文件。
3. 这些文件的父目录并不是当前已索引目录。
```

这时不应该强行把父目录整棵树加入索引，而是放入虚拟目录。

### 4.2 虚拟目录建议命名

建议使用一个清晰、统一的顶层节点名称，例如：

```text
1. 临时导入
2. 未归类文件
3. 导入文件
```

本文档建议默认使用：

```text
临时导入
```

### 4.3 虚拟目录组织方式

虚拟目录下建议再按原始来源目录分组：

```text
临时导入
├─ /Users/.../Downloads
│  ├─ a.md
│  └─ b.docx
└─ /Users/.../Desktop
   └─ c.html
```

这样既保留了“虚拟目录”这一层统一入口，也不丢失原始来源上下文。

---

## 5. 目录树规则

### 5.1 真实目录树

真实目录树仍然按路径层级组织：

```text
1. 父目录
2. 子目录
3. 叶子目录
```

### 5.2 虚拟目录树

虚拟目录是逻辑节点，不对应真实文件系统路径。

```text
1. 虚拟目录只存在于 UI 和导入结果中。
2. 不应被当成真实路径传给文件系统操作。
3. 虚拟目录下的条目仍然支持打开、重切片、查看详情。
```

### 5.3 目录树与导入结果合并

最终目录视图中需要同时展示：

```text
1. 真实目录树。
2. 虚拟目录。
```

两者在视觉上可以保持统一风格，但逻辑上要区分：

```text
1. 真实目录 -> 来自路径树。
2. 虚拟目录 -> 来自单文件导入归类。
```

## 6. 数据与协议建议

### 6.1 前端输入

拖拽事件需要向后端传递路径列表：

```text
1. directory paths
2. file paths
```

前端负责基础类型识别和 UI 状态展示。

### 6.2 后端识别

后端接收路径后需要识别：

```text
1. path 是目录还是文件。
2. 文件所属的父目录是否已存在于当前索引树。
3. 文件是否应归入虚拟目录。
```

### 6.3 建议返回结果

导入命令建议返回统计结果：

```text
1. added_dirs
2. added_files
3. grouped_to_virtual
4. skipped
5. unsupported
6. errors
```

这样前端可以在导入结束后给出更明确的反馈。

---

## 7. UI 交互建议

### 7.1 拖拽区域

在 `文档目录` 页提供明显的拖拽接收区：

```text
1. 支持拖入目录或文档。
2. 拖拽悬停时有高亮。
3. 松开后开始导入。
```

### 7.2 提示文案

拖拽时建议显示：

```text
1. 松开以添加目录
2. 松开以导入文档
3. 支持 md / docx / html / txt
```

### 7.3 导入完成反馈

导入结束后应自动刷新目录树和状态，不要求用户手动刷新。

```text
1. 显示新增目录 / 文件数量。
2. 显示进入虚拟目录的文件数量。
3. 显示跳过和不支持项。
```

---

## 8. 推荐实现顺序

```text
1. 前端先接拖拽入口。
2. 后端新增“导入路径”命令。
3. 后端识别目录 / 文件并返回统计。
4. 前端根据结果刷新目录树。
5. 最后再优化虚拟目录的树形展示。
```

---

## 9. 验收标准

当以下条件满足时，说明该方案完成：

```text
1. 可以直接拖入目录。
2. 可以直接拖入单个或多个文档。
3. 目录树能按路径层级展示。
4. 单文件导入不会误把整棵父目录树纳入索引。
5. 不属于真实目录树的文件会进入虚拟目录。
6. 导入完成后目录树自动刷新。
```
