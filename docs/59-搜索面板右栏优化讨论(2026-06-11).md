## 1. 右栏应该分成 4 个清晰区域

建议结构如下：

```

├─ 文件名 / 类型 / 路径 / 打开按钮

命中概览区
├─ 命中段落：第 1 段
├─ 命中类型：正文摘要命中
├─ 匹配范围：0–220 / 224
├─ 相似度 / 关键词

├─ 真正命中的文本，高亮关键词
├─ 只展示核心片段，不要混入调试字段

上下文区
├─ 上一段
├─ 当前段
├─ 下一段
```

现在你右栏里最大的问题是：  
`命中段落、上下文预览、当前段、标题路径、上一段、下一段、摘要范围` 全都混在同一个白底区域里，用户不知道哪里是“系统说明”，哪里是“文档原文”。

## 3. 信息样式要区分“标签”和“正文”

你现在的问题之一是：所有文字颜色、字号、间距差不多。

建议这样定：

| 内容类型 | 样式 |
| --- | --- |
| 文件名 | 16px / 600 / 深色 |
| 路径 | 12px / 灰色 / 单行省略 |
| 标签名 | 12px / 灰色 / 固定宽度 |
| 标签值 | 13px / 深色 |
| 命中状态 | 绿色 pill |
| 命中原文 | 14px / 行高 1.8 / 白底 |
| 上下文 | 13px / 灰蓝色 / 次级卡片 |
| 命中关键词 | 黄色浅底或蓝色浅底高亮 |

重点是：**标签文字不能和正文抢视觉权重。**

例如：

```
命中段落    第 1 段
命中类型    正文摘要命中
匹配范围    0–220 / 224
```

不要写成：

```
命中段落
Windows利用SSH...
上下文预览
当前段
标题路径：1.参考
```

## 4. “命中原文”要成为视觉中心

右栏最重要的不是元信息，而是命中的原文。建议命中原文使用独立卡片，并加一个明显标题：

```
命中原文
```

里面只放文档内容，不放系统字段。

比如：

```
Windows利用SSH - X远程登录Linux主机（X11-forwarding）

上下文预览：
当前段

Windows利用SSH - X远程登录Linux主机（X11-forwarding）
第 1 段
```

可以优化成：

```
Windows利用SSH - X远程登录Linux主机（X11-forwarding）

我们在 Microsoft Windows 平台通过 SSH 访问 X11，需要准备：
1. SSH 客户端，例如 PuTTY
2. X Server，例如 Xming
```

也就是说，不要把：

```
上下文预览
当前段
标题路径：1.参考
第 1 段
```

这些系统说明混进正文卡片里。

## 5. 上一段 / 当前段 / 下一段建议做成分段卡片

现在“上一段、当前段、下一段”挤在一起，很像日志。

建议改成：

```
上下文预览

[上一段]
无内容

[当前段]
Windows利用SSH - X远程登录Linux主机...

[下一段]
我们在 Microsoft Windows 平台通过 SSH 访问 X11...
```

视觉上可以这样：

```
┌ 上一段 ─────────────┐
│ 无内容              │
└────────────────────┘

┌ 当前段 ─────────────┐
│ Windows利用SSH...   │
└────────────────────┘

┌ 下一段 ─────────────┐
│ 我们在 Microsoft... │
└────────────────────┘
```

其中“当前段”可以加淡蓝色边框或浅蓝背景，让用户立刻知道它是核心命中段。

## 6. 可以增加“为什么命中”

右栏最好加一个很小的解释区，例如：

```
为什么命中
该段与搜索词 “WBS” 在语义上相关，命中字段为：正文摘要。
```

对于语义搜索产品，这个信息很重要。用户经常会疑惑：  
“为什么我搜 WBS，出来的是 ssh 文档？”

如果搜索结果不合理，右栏也应该能帮助用户判断是：

```
命中原因：语义相似
匹配字段：正文摘要
匹配分数：52.37
```

而不是直接把 `searchResultGroupCard.matchScore` 这种开发字段暴露出来。

## 8. 最关键的改动建议

你现在不需要大改功能，优先做这几个 UI 调整就会明显变好：

1. **把元信息和正文分离**
2. **命中原文单独成卡片**
3. **当前段高亮，上一段/下一段弱化**
4. **去掉开发字段，例如 `searchResultGroupCard.matchScore`**
5. **标题路径、匹配范围、切片数这些用表格式 label-value 展示**
6. **命中关键词做背景高亮**
7. **右栏文字整体加大行高，减少密集感**

右栏不要再像“搜索结果详情 JSON 的可视化”，而要像“用户正在阅读一段被命中的文档内容”。
