# 0.7x 未完成项收口方案

本文用于承接 `docs/30-0.7x完成清单vs当前代码差异清单(2026-06-03).md` 中列出的未完成项，给出 0.7x 收口的实现建议和优先级。

目标：让 `0.7x` 在不膨胀为完整笔记软件的前提下，完成主题集合的闭环收口。

---

## 1. 收口原则

```text
1. 只补 0.7x 完成定义里缺失的必要能力。
2. 不把推荐能力、复杂标签系统、富文本笔记做进当前版本。
3. 所有来源操作仍然优先回到原始文档、chunk 或 query。
4. 右键菜单优先，减少按钮堆叠。
5. 继续保持本地优先，数据只落 SQLite。
```

---

## 2. 未完成项与收口方向

### 2.1 文档级收藏入口

当前差异：

```text
当前只有 chunk / qa_source 入集合，没有明确的 document 级收藏入口。
```

收口方案：

```text
1. 在搜索页结果卡和右键菜单补“收藏文档”。
2. 在问答页来源摘要右键菜单补“收藏文档”。
3. 收藏落库时使用 item_type=document。
4. document 收藏只要求 document_id，其他字段保存文档快照。
5. 集合页对 document 类型使用独立展示语义，但不增加额外复杂编辑能力。
```

建议字段快照：

```text
title: 文件名
path: 完整路径
title_path: 可为空或保留摘要
snippet: 文档首段或简短摘要
source_meta_json: 保存来源上下文
```

完成标准：

```text
用户可以显式收藏文档，并在集合页回看该文档来源。
```

---

### 2.2 集合页标签过滤

当前差异：

```text
已有标签增删，但没有集合内按标签过滤。
```

收口方案：

```text
1. 在集合页中间栏顶部增加 tag chips。
2. 默认先做单选过滤，避免交互过重。
3. 筛选范围只作用于当前 collection_items。
4. 筛选结果仍保留右键菜单和来源详情能力。
```

交互要求：

```text
1. 标签可点击切换筛选。
2. 选中标签时高亮。
3. 允许一键清除筛选。
```

完成标准：

```text
用户可以在单个集合内按标签快速定位来源。
```

---

### 2.3 最近查看回流入口

当前差异：

```text
已有最近查看列表，但还没有形成强回流入口。
```

收口方案：

```text
1. 保留最近查看列表，不扩展成复杂历史中心。
2. 最近查看只覆盖 document / collection / qa_session 三类。
3. 点击即可回到对应页面。
4. 只在知识库页或侧边栏保留轻量入口，不新增独立历史页。
```

完成标准：

```text
用户可以快速回到最近查看的文档、集合或问答会话。
```

---

### 2.4 推荐能力后移

当前差异：

```text
相似 chunk 推荐、相关文档推荐、解释性推荐未实现。
```

收口方案：

```text
1. 不纳入 0.7x 收口。
2. 单独进入后续版本规划，如 0.8x 或更后续阶段。
3. 保持 0.7x 只做“可沉淀来源”，不做“自动推荐关系”。
```

原因：

```text
推荐能力会显著增加解释、排序和 UI 复杂度，不是 0.7x 的必要完成条件。
```

---

## 3. 建议实施顺序

```text
1. 补文档级收藏入口。
2. 补集合页标签过滤。
3. 收口最近查看入口。
4. 明确将推荐能力移出 0.7x。
```

---

## 4. 0.7x 收口完成定义

当以下条件都满足时，可以把 `0.7x` 标记为完成：

```text
1. 可以创建、重命名、删除主题集合。
2. 可以收藏文档、段落和问答来源。
3. 可以把搜索结果和问答来源加入主题集合。
4. 可以在集合页回顾来源并打开原文件。
5. 可以给集合条目添加备注。
6. 可以导出集合引用列表。
7. 可以按标签筛选集合内容。
8. 不影响现有搜索、语义搜索、问答和来源预览能力。
```

---

## 5. 不纳入本次收口的内容

```text
1. 复杂推荐。
2. 富文本笔记编辑器。
3. 双链、反链、知识图谱。
4. 团队协作和云同步。
5. 复杂标签管理中心。
```
