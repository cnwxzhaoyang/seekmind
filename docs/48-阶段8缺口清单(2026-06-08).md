# 阶段 8 缺口清单：性能、打包和商业化准备

本文档是 [《阶段 8 对照表》](./47-阶段8对照表(2026-06-08).md) 的配套清单，聚焦当前代码库在发布准备层面的主要缺口。

## 已确认缺口

### 1. 性能基线仍未完整

现状：
- 代码里已经有索引、语义和 OCR 主链路。
- 已补 `seekmind-benchmark`，可以输出搜索与运行时基线 JSON 报告。

缺口：
- 缺少 1,000 / 5,000 / 10,000 文档分档基线数据。
- 缺少全量索引、增量索引、恢复索引的耗时记录。
- 当前内存指标只有进程峰值 RSS，仍不够细。

建议：
- 保留当前 benchmark 作为第 1 层基线。
- 下一步直接补三档测试集和批量压测脚本。

### 2. 大文档库压测仍需补强

现状：
- 已补 `seekmind-stress`，可以生成临时大库并执行全量重建。

缺口：
- 还缺更接近真实用户的数据结构和体量样本。
- 还缺增量索引、恢复索引和持续压测模式。

建议：
- 先把 10,000 文档样本跑通，再补 1,000 / 5,000 / 10,000 三档对比报告。
- 后续把增量与恢复模式独立出来。

### 3. 索引版本迁移机制已补齐

现状：
- 数据库 schema 已支持增量迁移。
- 全文索引已支持独立 schema 版本标记。
- 语义向量索引已支持 `vector_index_meta.schema_version` 与失配后自动重建。

结论：
- 阶段 8 的“索引版本迁移机制”现已完成，不再作为当前缺口。
- 后续若新增全文字段、向量维度或 chunk 结构，只需沿当前版本号和重建路径继续扩展。

### 4. 崩溃恢复仍停留在索引任务层

现状：
- 已有 checkpoint 和 `resume_indexing`。

缺口：
- 缺少更完整的崩溃恢复说明与恢复入口。
- 缺少应用异常退出后的恢复提示。

建议：
- 将现有 checkpoint 能力提升为用户可感知的“恢复未完成任务”能力。

### 5. 日志导出已补齐

现状：
- 已有底部日志面板。
- 已支持从日志面板导出当前事件、运行摘要和环境信息到本地 Markdown 文件。

结论：
- 阶段 8 的“日志导出”现已完成，不再作为当前缺口。
- 后续如需扩展，可以再补“完整诊断包”或“后台历史日志收集”。

### 6. 自动更新缺失

现状：
- 已补 updater 基础配置、版本检查和设置页更新源配置。

缺口：
- 没有完整的正式发布服务接入。
- 没有静默下载和自动安装闭环。
- 还没有把更新源、签名验证和灰度策略串成生产级流程。

建议：
- 先补正式发布源和下载入口。
- 再评估静默下载、自动安装和升级后重启提示。
- 最后补更新签名、灰度和失败回退策略。

### 7. 首次启动引导已补齐

现状：
- 已在桌面壳层加入轻量首次使用引导条。
- 当尚未添加索引目录时，会提示用户先添加目录，并可一键跳转到索引状态页。

结论：
- 阶段 8 的“首次启动引导”现已完成，不再作为当前缺口。
- 后续如需扩展，可以再补更完整的首次向导或新手教程页。

### 8. 隐私说明还不够成产品交付

现状：
- 设置页已有局部隐私说明。

缺口：
- 缺少完整的隐私说明文案与入口。
- 缺少对“本地索引 / 外部模型 / 网络代理 / 日志内容”边界的成文说明。

建议：
- 增加独立的隐私说明页或帮助页。

### 9. License / Pro 功能开关缺失

现状：
- 当前未看到授权、试用或 Pro gating 逻辑。

缺口：
- 缺少商业化最小框架。
- 缺少 feature flag 体系和 UI 呈现方式。

建议：
- 先做本地 feature flag / Pro 开关架构。
- 激活、联网授权和试用机制放到第二步。

## 推荐补齐顺序

1. 性能基线与压测
2. 自动更新
3. 隐私说明
4. License / Pro 功能开关
5. 崩溃恢复产品化收口

## 结论

阶段 8 当前的主要问题不是“完全没有基础”，而是：
- 缺基础指标
- 缺发布闭环
- 缺商业化骨架

所以阶段 8 的重点不应该先做零散功能，而应该先建立“发布前最小基础设施”。
