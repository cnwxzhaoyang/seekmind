# 阶段 8 首次启动引导

本文档记录阶段 8 中“首次启动引导”的实现结果，作为新用户的最小上手路径。

## 1. 已实现内容

- 在全局桌面壳层加入轻量首次使用引导条。
- 当当前还没有任何索引目录时，引导条会自动出现。
- 引导条提供跳转到索引状态页的入口，帮助用户先添加目录再开始使用。
- 用户可以手动关闭引导条，关闭后会在本地记忆，不再重复出现。

## 1.1 开发环境模拟命令

```bash
npm run tauri:dev:first-launch
```

该命令会在启动前清空本地存储，并强制开启首次启动引导，适合开发和联调时反复复现。

## 2. 引导内容

引导条包含三个关键步骤：
- 添加索引目录
- 等待首次索引
- 开始搜索与问答

## 3. 设计取舍

- 没有做全屏遮罩或长教程，避免打断桌面应用的直接可用性。
- 引导采用顶部轻量横条，用户可以忽略、关闭，也可以直接跳到索引状态页。
- 只有在未配置索引目录时才显示，减少对老用户的打扰。

## 4. 实现位置

- 引导组件：`src/components/SeekMind/SeekMindFirstLaunchGuide.vue`
- 全局壳层挂载：`src/components/SeekMind/SeekMindPageShell.vue`
- 首次状态读取：`src/services/seekMindApi.ts`

## 5. 验证结果

- `cargo check --manifest-path src-tauri/Cargo.toml` 通过
- `npm run build` 通过

## 6. 结论

- 阶段 8 的“首次启动引导”已完成。
- 当前阶段 8 的重点继续转向自动更新、隐私说明和商业化骨架。
