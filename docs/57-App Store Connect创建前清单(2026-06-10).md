# App Store Connect 创建前清单

本文档用于在进入 App Store Connect 之前，快速确认当前项目需要准备的字段、素材和发布前置条件。  
目标不是提审，而是先把 App 记录建起来，并避免后续反复补材料。

## 1. 当前判断

当前项目已经可以开始创建 App Store Connect 的 App 记录，原因是：

- 已有稳定的桌面端应用壳层和主功能入口。
- 已补自动更新基础链路。
- 已有日志导出、首次启动引导、索引版本迁移等发布准备能力。
- 已支持中英文切换。

但还不建议直接提审，因为正式发布闭环仍需补齐：

- 生产级更新源。
- 静默下载和自动安装闭环。
- 最终签名、截图和元数据校验。
- 隐私政策与支持页的正式地址。

## 2. 创建 App 前需要准备的字段

### 2.1 基础信息

必须准备：

- App 名称
- Bundle ID
- SKU
- 主分类
- 支持网址
- 隐私政策网址

建议准备：

- 副标题
- 关键字
- 版权信息
- App 简介

### 2.2 建议值

结合当前项目状态，建议如下：

- App 名称：`SeekMind`
- SKU：使用内部稳定标识，例如 `seekmind-desktop` 或 `seekmind-app`
- 主分类：`Productivity` 或 `Utilities`
- 支持网址：项目官网或产品落地页
- 隐私政策网址：独立可访问页面

如果暂时没有官网或落地页，不建议填临时链接，后续容易在审核阶段来回修改。

## 3. 需要准备的素材

### 3.1 图标

需要准备：

- 1024x1024 App 图标
- 各尺寸 App Store 图标资源

建议检查：

- 是否与桌面端当前品牌一致
- 是否在深色背景和浅色背景下都清晰

### 3.2 截图

建议至少准备：

- 启动页或首页截图
- 搜索页截图
- 问答页截图
- 知识库页截图
- 索引状态页截图
- 设置页截图

建议同时准备：

- 深色主题截图
- 浅色主题截图

### 3.3 文案

需要准备：

- App 简介
- 功能说明
- 更新说明模板
- 隐私政策摘要

## 4. 技术侧检查项

创建 App 记录前，建议先确认以下技术项：

- Bundle ID 已稳定，不要再频繁变更。
- Tauri 打包配置已经固定。
- `auto-update` 的基础链路已存在，但正式发布源还未接入。
- 深色主题和浅色主题都已做基础适配。
- 语言切换已可用，中英文都能跑通。

## 5. 当前项目的已准备项

目前可以直接写进 App 记录或作为审核前基础材料的部分包括：

- 本地索引与检索主流程
- 问答页
- 知识库页
- 索引状态页
- 自动更新基础链路
- 日志导出
- 首次启动引导
- 中英文切换

## 6. 当前还不建议写死的内容

这些内容目前建议先保留为占位或后续补齐：

- 正式版本号策略
- 正式更新源地址
- 生产环境下载链接
- 隐私政策正式 URL（如果还未上线）
- 支持页正式 URL（如果还未上线）

原因是这些字段一旦写死，后续补正式发布闭环时容易再改一轮元数据。

## 7. 创建顺序建议

建议按下面顺序操作：

1. 先确认 Bundle ID 和 SKU。
2. 再准备支持网址和隐私政策网址。
3. 然后准备图标与截图。
4. 最后在 App Store Connect 里创建 App 记录。

## 8. 风险提示

- 不要在 Bundle ID 还不稳定时创建多个重复 App。
- 不要把临时测试地址当成正式支持页或隐私页。
- 不要在自动更新还没接入正式发布源时就把文案写成“已完整支持自动更新”。
- 不要在截图里展示与当前版本不一致的 UI。

## 9. 后续动作

创建 App 记录后，下一步建议继续做：

- 补正式发布源
- 补自动更新完整闭环
- 补隐私政策页
- 补 App Store 截图与元数据文案
