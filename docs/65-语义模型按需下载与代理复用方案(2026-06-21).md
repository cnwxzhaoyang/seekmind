# 65-语义模型按需下载与代理复用方案(2026-06-21)

## 背景

当前 SeekMind 的语义模型倾向于随安装包一起分发。作为准备上架的桌面 App，这会带来几个问题：

1. 安装包体积变大，影响下载、审核和用户首次安装体验。
2. 用户没有机会按自己的网络、磁盘和机器性能选择模型。
3. 模型升级需要跟随 App 发版，灵活性不足。
4. 不同地区用户访问模型源的体验差异较大，需要复用已有网络代理能力。

因此，语义模型应从“随包内置”调整为“按需下载”。App 只内置模型下载能力和默认模型清单地址，用户在设置页自行选择是否下载、下载哪个模型、是否使用代理。

## 当前实现进度

2026-06-21 已完成第一段落地：

1. Tauri 打包资源不再整体打入 `app-resources`，改为只打入 OCR 和 parser 运行资源，避免本机残留的 `app-resources/fastembed` 进入安装包。
2. Rust 侧新增语义模型下载模块，支持模型清单占位、下载、SHA-256 校验、解压、防路径穿越检查、删除本地模型。
3. 下载请求复用现有网络代理配置。
4. 前端语义模型设置页新增“模型下载”列表，展示下载状态、源未配置状态、下载进度、删除本地模型入口。
5. 运行时不再从安装包资源自动恢复 FastEmbed 模型，缺失时等待用户在设置页按需下载。
6. 第一阶段下载源使用 GitHub Release：
   - 仓库：`cnwxzhaoyang/seekmind`

   - Size：`55371779`

仍保留为后续阶段：

1. 远程模型清单服务。
2. 断点续传、暂停、取消。
3. 多模型选择和设为默认模型的完整联动。
4. 正式下载源 URL 与 SHA-256 发布流程。

## GitHub Release 临时发布流程

第一阶段可以先用 GitHub Release 承载模型包。

本地生成模型包：

```bash
npm run semantic:warmup:mirror
npm run semantic:package
```

产物位置：

```text
.seekmind-build/semantic-models/fastembed-cache.tar.gz
.seekmind-build/semantic-models/fastembed-cache.tar.gz.sha256
```

使用 GitHub CLI 上传：

```bash
gh release create semantic-models-v1 \
  .seekmind-build/semantic-models/fastembed-cache.tar.gz \
  .seekmind-build/semantic-models/fastembed-cache.tar.gz.sha256 \
  --repo cnwxzhaoyang/seekmind \
  --title "Semantic Models v1" \
  --notes "FastEmbed cache for BGE Small zh v1.5."
```

如果 release 已存在，使用：

```bash
gh release upload semantic-models-v1 \
  .seekmind-build/semantic-models/fastembed-cache.tar.gz \
  .seekmind-build/semantic-models/fastembed-cache.tar.gz.sha256 \
  --repo cnwxzhaoyang/seekmind \
  --clobber
```

上传后必须确认代码里的 URL、SHA-256、Size 与 release asset 完全一致，否则下载会在校验阶段失败。

## 目标

1. 安装包默认不打入大体积语义模型。
2. 设置页提供模型选择、下载、暂停/取消、删除和设为默认能力。
3. 模型清单和模型文件下载均复用现有网络代理配置。
4. 下载完成后必须做文件完整性校验，避免损坏模型进入运行链路。
5. 支持后续替换下载源、增加镜像源、下架模型，而不强依赖 App 发版。

## 不做内容

第一阶段不做：

1. 付费模型市场。
2. 用户自定义任意 Hugging Face 仓库解析。
3. 自动后台下载大模型。
4. 模型增量更新。
5. 云端推理服务。

## 总体架构

推荐将语义模型能力拆成四层：

1. 模型清单：描述可下载模型、版本、大小、下载地址、校验值和兼容性。
2. 下载管理：负责清单拉取、模型下载、断点续传、取消、进度和错误状态。
3. 本地模型仓库：管理已下载模型、默认模型、磁盘路径和删除。
4. 语义检索运行时：只依赖本地已就绪模型，不关心下载来源。

流程：

```text
设置页打开
  -> 拉取模型清单
  -> 展示可选模型
  -> 用户点击下载
  -> 下载器读取网络代理配置
  -> 下载模型文件到临时目录
  -> 校验 sha256
  -> 移动到模型目录
  -> 写入本地模型状态
  -> 用户设为默认模型
  -> 语义索引使用该模型重建或继续索引
```

## 下载源选择

### 推荐顺序

正式上架建议优先使用“对象存储 + CDN”，同时保留公共镜像源作为备选。

推荐顺序：

1. 对象存储 + CDN
2. GitHub Releases
3. Hugging Face / ModelScope
4. 自建服务器

### 对象存储 + CDN

适合正式版本：

- 可控性强，可以下架、替换、灰度和加镜像。
- 下载速度和稳定性可管理。
- 可按地域选择国内/海外 CDN。
- 适合较大模型文件。

成本主要来自：

- 对象存储容量
- 下载流量
- CDN 流量
- HTTPS 和域名维护

### GitHub Releases

适合早期验证：

- 成本低，上手快。
- 适合放公开模型文件或测试模型。

限制：

- 国内访问不稳定。
- 大规模下载可能遇到限流或速度问题。
- 商业上架时不可控因素较多。

### Hugging Face / ModelScope

适合引用开源模型来源：

- 可以清晰表达模型来源。
- 对模型社区用户友好。

限制：

- 平台访问稳定性不可控。
- URL、鉴权、限流策略可能变化。
- 不宜作为唯一正式下载源。

### 自建服务器

不建议直接承担模型文件下载。

如果使用自建服务，建议只承载模型清单和调度逻辑，大文件仍放对象存储或 CDN。

## 模型清单设计

App 不应写死单个模型下载地址，而应内置一个模型清单地址。

清单示例：

```json
{
  "version": 1,
  "updated_at": "2026-06-21",
  "models": [
    {
      "id": "bge-small-zh-v1.5-onnx",
      "name": "BGE Small 中文",
      "provider": "BAAI",
      "version": "1.5",
      "runtime": "onnx",
      "language": ["zh", "en"],
      "dimension": 512,
      "size_bytes": 102400000,
      "recommended": true,
      "description": "体积小，适合普通电脑，中文文档检索优先推荐。",
      "files": [
        {
          "name": "model.onnx",
          "url": "https://cdn.example.com/seekmind/models/bge-small-zh-v1.5/model.onnx",
          "sha256": "待填充",
          "size_bytes": 102400000
        },
        {
          "name": "tokenizer.json",
          "url": "https://cdn.example.com/seekmind/models/bge-small-zh-v1.5/tokenizer.json",
          "sha256": "待填充",
          "size_bytes": 1200000
        }
      ],
      "mirrors": [
        {
          "name": "GitHub Releases",
          "base_url": "https://github.com/example/seekmind-models/releases/download/bge-small-zh-v1.5"
        },
        {
          "name": "ModelScope",
          "base_url": "https://modelscope.cn/models/example/bge-small-zh-v1.5/files"
        }
      ],
      "min_app_version": "0.1.0",
      "platforms": ["macos", "windows"]
    }
  ]
}
```

字段说明：

- `id`：模型唯一标识，后续本地状态和默认模型引用都使用它。
- `runtime`：运行格式，例如 `onnx`。
- `dimension`：向量维度，用于校验与向量库兼容性。
- `files`：模型实际需要下载的文件列表。
- `sha256`：每个文件必须提供，用于下载完成校验。
- `mirrors`：备用镜像源，下载失败时可切换。
- `min_app_version`：避免旧版本 App 下载不兼容模型。
- `platforms`：控制模型是否支持当前系统。

## 网络代理复用

现有设置页已经有网络代理配置，模型下载必须复用同一套配置。

需要代理的请求：

1. 模型清单拉取。
2. 模型文件 HEAD 请求。
3. 模型文件下载。
4. 断点续传请求。
5. 镜像源探测。

行为规则：

- 代理关闭：直接请求下载源。
- 代理开启：所有模型下载相关请求均走代理。
- 代理不可用：提示用户代理连接失败，并允许关闭代理后重试。
- 下载源不可达：提示下载源不可达，并允许切换镜像。
- 校验失败：删除临时文件，提示文件校验失败，可重试。

错误状态建议：

| 状态 | 说明 | 用户动作 |
| --- | --- | --- |
| `manifest_unreachable` | 模型清单不可达 | 检查网络或代理 |
| `proxy_unreachable` | 代理不可用 | 修改或关闭代理 |
| `download_failed` | 模型文件下载失败 | 重试或切换镜像 |
| `checksum_failed` | sha256 校验失败 | 重新下载 |
| `disk_full` | 磁盘空间不足 | 清理空间或更换目录 |
| `model_incompatible` | 模型与当前版本不兼容 | 升级 App 或选择其他模型 |

## 本地存储目录

模型文件应放在 App 数据目录下，不要放在安装包目录。

建议目录：

```text
app_data/
  models/
    manifest.json
    bge-small-zh-v1.5-onnx/
      model.onnx
      tokenizer.json
      model.meta.json
    downloads/
      bge-small-zh-v1.5-onnx.tmp/
```

说明：

- `downloads` 用于临时下载，下载完成并校验后再原子移动。
- `model.meta.json` 记录来源、版本、sha256、下载时间和文件大小。
- 删除模型时只删除对应模型目录，不影响索引目录和用户文档。

Windows 和 macOS 兼容要求：

- 所有路径拼接必须使用系统路径 API。
- 数据库存储模型路径时应保存规范化路径或模型 id，不依赖硬编码分隔符。
- 比较路径前必须统一规范化，避免 Windows 下 `\` 和 `/` 导致状态识别失败。

## 下载完整性与安全

最低要求：

1. 下载前检查磁盘剩余空间。
2. 下载到临时目录，不直接覆盖已可用模型。
3. 下载完成后逐文件计算 `sha256`。
4. 校验成功后再移动到正式模型目录。
5. 校验失败必须删除临时文件。
6. 清单必须通过 HTTPS 拉取。

后续增强：

- 模型清单签名。
- 文件签名。
- 官方下载源证书固定。
- 下载重试退避。
- 下载限速。

## 设置页交互建议

设置页中的语义模型卡片可以调整为：

1. 当前模型状态
   - 未下载
   - 下载中
   - 已下载
   - 校验失败
   - 不兼容

2. 模型列表
   - 模型名称
   - 体积
   - 语言
   - 推荐标签
   - 下载/暂停/取消/删除/设为默认

3. 下载源
   - 默认官方源
   - GitHub Releases
   - ModelScope
   - 自定义镜像源

4. 网络代理提示
   - 当代理开启时展示“下载将使用网络代理”
   - 当下载失败且代理关闭时提示“可在网络代理中配置代理后重试”

不建议把下载地址裸露给普通用户作为必填项。下载地址应来自模型清单；高级用户可以后续再提供自定义清单地址。

## 数据结构建议

本地可以新增模型状态表：

```sql
CREATE TABLE IF NOT EXISTS semantic_models (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    version TEXT NOT NULL,
    runtime TEXT NOT NULL,
    dimension INTEGER NOT NULL,
    status TEXT NOT NULL,
    local_dir TEXT NOT NULL DEFAULT '',
    source_url TEXT NOT NULL DEFAULT '',
    sha256_json TEXT NOT NULL DEFAULT '',
    size_bytes INTEGER NOT NULL DEFAULT 0,
    downloaded_at TEXT NOT NULL DEFAULT '',
    updated_at TEXT NOT NULL DEFAULT ''
);
```

默认模型可以放在设置表：

```sql
semantic_default_model_id TEXT NOT NULL DEFAULT ''
```

也可以单独使用 key-value 设置表，避免频繁改主设置表结构。

## 实施步骤

### 第一阶段：模型清单与状态展示

1. 定义模型清单 JSON 结构。
2. 设置页新增模型列表。
3. 拉取清单时复用网络代理配置。
4. 展示模型大小、推荐状态和兼容状态。

### 第二阶段：下载与校验

1. 实现模型文件下载。
2. 下载进度通过事件推送到前端。
3. 下载到临时目录。
4. 完成后校验 `sha256`。
5. 校验成功后移动到正式模型目录。

### 第三阶段：本地模型管理

1. 写入模型状态表。
2. 支持删除模型。
3. 支持设置默认模型。
4. 语义索引运行时读取默认模型。

### 第四阶段：镜像源与失败恢复

1. 下载失败后允许切换镜像。
2. 支持断点续传。
3. 支持取消下载。
4. 支持清理失败临时文件。

### 第五阶段：发布源治理

1. 正式模型文件迁移到对象存储 + CDN。
2. 保留 GitHub Releases / ModelScope 作为备用镜像。
3. 模型清单加入签名或版本校验。

## 推荐落地策略

短期：

- 不再把大模型打入安装包。
- 使用 GitHub Releases 或 ModelScope 验证按需下载流程。
- 设置页明确提示可使用网络代理下载。

正式上架前：

- 准备对象存储 + CDN 作为官方源。
- 提供至少一个备用镜像源。
- 完成 sha256 校验、失败重试和本地模型删除。

长期：

- 支持自定义模型清单地址。
- 支持模型清单签名。
- 支持根据设备性能推荐模型。

## 风险与对策

| 风险 | 对策 |
| --- | --- |
| 下载源不可达 | 支持代理和镜像源 |
| 模型文件损坏 | sha256 校验，失败删除临时文件 |
| 用户磁盘不足 | 下载前检查空间，错误提示可恢复 |
| 模型与索引不兼容 | 记录 dimension 和 model_id，切换模型后提示重建语义索引 |
| 国内外网络差异 | 官方源使用 CDN，公共平台作为备用 |
| App 更新与模型更新耦合 | App 内置清单地址，模型通过清单独立发布 |

## 结论

模型下载不要求一开始就购买外部云服务。早期可以用 GitHub Releases、Hugging Face 或 ModelScope 验证流程。

但作为正式上架 App，建议最终准备一个官方可控下载源，优先选择对象存储 + CDN。现有网络代理配置应贯穿模型清单拉取和模型文件下载，作为用户处理特殊网络环境的第一入口。
