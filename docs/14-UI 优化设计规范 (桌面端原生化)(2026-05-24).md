# SeekMind UI 优化设计规范 (桌面端原生化)

## 1. 现状分析
*   **Web 感过强**：大面积的圆角卡片、宽大的边距和大量的空白区域是典型的现代 Web Dashboard 风格，但在桌面端会显得不够高效。
*   **空间利用率低**：顶部统计信息占据了过多垂直空间，导致实际搜索结果区域受限。
*   **层级不明确**：侧边栏、内容区和底部的日志区界限较为模糊，缺乏原生应用的沉浸感。

## 2. 优化目标
*   **紧凑 (Compact)**：减少冗余边距，提高信息展示密度。
*   **简洁 (Minimal)**：合并同类项，隐藏非核心信息，突出搜索和结果。
*   **原生感 (Native Feel)**：采用多栏布局（Multi-pane），使用系统字体，增强视觉稳定性。

## 3. 视觉规范建议

### 布局 (Layout)
*   **左侧导航栏**：宽度缩减，使用图标 + 文字的组合，背景色与主内容区形成微弱对比。
*   **顶部搜索区**：不再作为独立的大卡片，而是作为主内容区的 Header，集成搜索状态和过滤选项。
*   **中间统计/快捷区**：将原本巨大的卡片缩小为一行或折叠式面板，或者移至侧边栏下方。
*   **主结果区**：采用列表流式布局，支持两栏或三栏设计（列表 + 预览）。

### 色彩与材质 (Color & Material)
*   **背景**：主背景使用浅灰色或纯白（支持暗黑模式），侧边栏使用微弱的磨砂玻璃感（Acrylic/Vibrancy）。
*   **强调色**：保留现有的蓝色/深紫色作为动作色，但减小其应用面积。
*   **边框**：使用极细的边框（1px）或阴影来区分区域，而非大圆角卡片。

### 字体 (Typography)
*   **系统字体栈**：优先使用 `Inter`, `Segoe UI` (Windows) 或 `San Francisco` (macOS)。
*   **字号**：正文字号保持在 12px-14px，标题 16px，减少大字号的使用。

## 4. 交互优化
*   **搜索即时性**：输入时实时更新状态，而非点击“搜索”按钮。
*   **快捷键支持**：UI 中增加快捷键提示（如 `Cmd+K` 搜索，`Cmd+1, 2...` 切换模块）。
*   **状态反馈**：将“已索引 50 个文档”等状态信息移至底部状态栏（Status Bar）。
